# Google Sheets 資料結構

本文件描述系統使用的三種試算表格式，內容全部為**虛構示範資料**
（來源 demo xlsx 已被 `.gitignore` 的 `*.xlsx` 排除，不進版控）。
欄列語意均對照 `src/Code.js` 實作（`getQList_`、`getHeaders`、`writeRecord` 等）驗證過。

## 三種試算表的角色

| 試算表 | 取得方式 | 用途 |
|--------|----------|------|
| 問卷列表（listSS） | ScriptProperties `listSheetID` | 所有問卷的總目錄，一列一份問卷 |
| 對照表單（referSS，名冊） | 問卷列表 B 欄「對照表單ID」 | 定義欄位（前 8 列）＋名冊資料（第 9 列起） |
| 紀錄表單（recordSS） | 問卷列表 C 欄「新表單ID」 | 填寫結果，`writeRecord` 一次送出 append 一列 |

另有 ScriptProperties：`draftSheetID`（線上暫存＋`_invites` 邀請分頁）、
`universalStorageID`（上傳檔案／簽名圖的 Drive 資料夾）、`emailLog`、`jwtSecret`、`systemTitle`。
暫存與邀請分頁結構見 `struct.md`，不在本文件範圍。

## 1. 問卷列表（listSheetID 指向的試算表）

Web app 只讀**第一個分頁**的 `A:P`（16 欄），第 1 列為標題列，第 2 列起一列一份問卷。
（實際檔案裡可能還有「輸出主控台」「執行紀錄」等分頁，那是 `tools/` 管理工具在用的，web app 不讀。）

| 欄 | 標題 | `getQList_` 對應 | 說明 |
|----|------|------------------|------|
| A | 表單名稱 | `name` | 空字串＝整列略過 |
| B | 對照表單ID | `refer` | 名冊試算表的 Sheet ID |
| C | 新表單ID | `record` | 紀錄試算表的 Sheet ID |
| D | 填表截止日期Timestamp | `dueDate` | 毫秒 timestamp；過期後 `enableModify=false`、清空簽名需求 |
| E | 檢視截止時間Timestamp | `viewDate` | 毫秒 timestamp；過期後列表上完全不顯示 |
| F | 預設修改 | `enableModify` | 「否」＝false，其他＝true |
| G | 簽名 | `signatures` | 需要的簽名格名稱，`;` 分隔（如 `家長;導師`），空＝不用簽名 |
| H | 登入後說明 | `comment` | Markdown，前端過 DOMPurify |
| I | 登入前說明 | `loginTip` | 同上 |
| J | 填寫完畢備註語 | `submitTip` | 同上 |
| K | 登入失敗備註語 | `loginfailTip` | 同上 |
| L | 顯示 | — | 「是」才會進列表，其他值整列略過 |
| M | 管理員Email | `email` | 送出通知信收件人 |
| N | 固定ID | `sheetID` | `?sheet=` 直連參數用的固定代號 |
| O | 開放進入 | `writeAllowed` | 「是」＝可登入填寫；否則唯讀 |
| P | 亂數出題 | `randomQ` | 「是」＝欄位順序隨機 |

### 範例列（虛構）

| 表單名稱 | 對照表單ID | 新表單ID | 填表截止 | 檢視截止 | 預設修改 | 簽名 | … | 顯示 | 管理員Email | 固定ID | 開放進入 | 亂數出題 |
|---|---|---|---|---|---|---|---|---|---|---|---|---|
| [測試中] 測試問卷 | `1ReferSheetIdFakeAAAAAAAAAAAAAAAAAAAAAAAAAAA` | `1RecordSheetIdFakeBBBBBBBBBBBBBBBBBBBBBBBBB` | `1783726200000` | `1783726200000` | 是 | 測試者一;測試者二 | … | 是 | `admin@example.com` | 16 | 是 | （空） |

## 2. 對照表單／名冊（refer，「對照表單ID」指向）

只讀第一個分頁。**前 8 列是欄位定義（meta），第 9 列起是名冊資料**，一欄一個題目。
對應 `getHeaders()`：

| 列 | 內容 | 說明 |
|----|------|------|
| 1 | 欄位 ID | 如 `G01`、`P01`、`T01`，紀錄表與計算欄引用都靠它 |
| 2 | 欄位名稱 | 顯示給填寫者的標題 |
| 3 | type | 欄位角色，見下表 |
| 4 | format | 資料格式，見下表 |
| 5 | group | 群組標籤；同標籤的欄位「不得全空」（如三支電話欄都填「電話三擇一」）。後綴 `:U`（如 `電話三擇一:U`）＝群組內值不得重複——程式有實作（`columnPrep`／`validateColumn`），但實務問卷從未使用過 |
| 6 | content | 欄位參數，格式依 format 而異，見下表 |
| 7 | must | `M` ＝必填（regex test，容忍組合） |
| 8 | nullable | 詞彙可組合（`N`／`D`／`ND`）：`N` ＝可空（空值跳過格式檢查，紀錄落地空字串）；`D` ＝可宣告「無資料」（Phase 15）——前端出雙態按鈕一鍵填哨兵字串「無資料」，過必填、跳格式檢查、**原樣寫進紀錄**（下游可區分「宣告沒有」與「留白」），群組檢查中視同空值。`M`+`D` ＝必答但「無資料」是合法答案。**標 `D` 前務必先部署 Phase 15 後端**（見 issue.md） |
| 9+ | 名冊資料 | 一人一列：認證欄放名冊值、填寫欄放預設值、內容展示欄放要給該人看的內容 |

### type 代碼（第 3 列）

| type | 意義 |
|------|------|
| `P` | 主鍵（登入帳號）；`format=G` 時改走 Google 帳號驗證 |
| `A` | 認證欄位：登入時須與名冊列的值完全相符 |
| `O` | 輸出用欄位（如姓名）：登入後顯示、不可寫入（前端傳來也直接忽略） |
| `G` | 分組欄位（如班級）：`format=G` 的值會寫進紀錄列第 5 欄；`format=N` 用於名冊比對（座號） |
| `F` | 填寫欄位：真正要作答的題目 |
| `C` | 唯讀展示欄，依 format 分四種：`T` 展示名冊列該欄的文字、`M` 說明區塊（el-alert，content 為 markdown）、`S` 計算欄、`F` 檔案檢視（見下表） |

### format 代碼（第 4 列）與 content 參數（第 6 列）

| format | 意義 | content 格式（範例） |
|--------|------|----------------------|
| `T` | 文字輸入 | 空＝任意文字；或 `提示語::正則`（如 `必須要有號::號`、`格式：「20230101」::^(19|20)\d{2}…$`）。「無則填無」類題目慣用 `^(?:實際格式|無)$`。type=C 時＝展示名冊列該欄的文字，content 留空 |
| `X` | 長文字（textarea） | `最長字數;最少字數;顯示最小列;顯示最大列`（如 `100;3;4;4`） |
| `N` | 數字 | 長度（`6`＝6 碼）；`0`＝0 開頭、長度不限 |
| `I` | 身分證字號 | — （驗證 `^[A-Z][0-9A-Z]\d{8}$`） |
| `M` | 台灣手機 | — （驗證 `^09\d{8}$`）。type=C 時＝說明區塊：content 為 markdown 全文（標題取欄位名稱，經 DOMPurify） |
| `E` | Email | — |
| `P` | 郵遞區號 | `碼數;地址欄位ID,ID,…`（如 `3;F01,F02,F03`）：前端「自動填入郵遞區號」按鈕會把列出的地址欄位值串起來丟後端 `queryPC` 查出 3／6 碼 |
| `L` | 滑桿（整數） | `步進;最小;最大`（如 `1;0;10`） |
| `S` | 下拉單選 | `選項1;選項2;…`；**空＝自動取名冊第 9 列起該欄的唯一值當選項**（`buildSelections`）。type=C 時是計算欄：`欄位ID:倍數;…`（如 `N01:80;N02:180`＝各欄值×倍數加總） |
| `U` | 多選 | `數量::選項1;選項2;…`（如 `3::杏仁;葡萄;牛奶;巧克力;核桃`；數量空＝不限，如 `::無;心臟病;糖尿病;…`） |
| `F` | 檔案上傳 | `類型說明;允許副檔名(|分隔);大小上限MB;範例檔fileID`（如 `PDF或JPG圖檔;pdf|jpg|jpeg;5;1DriveFileIdFake…`）。type=C 時是**檔案檢視**：content 放 Drive **資料夾** ID（空＝universalStorageID），以名冊列該欄的值當檔名關鍵字在資料夾內搜尋，顯示找到的檔案（例：學生照片欄，名冊值＝身分證號，照片檔名含身分證號） |
| `G` | Google 帳號（type=P 登入用）／分組值（type=G） | — |

### 實務用法補充（取自真實問卷結構，值已虛構）

- **計算欄（C-S）配滑桿**：三個 `L` 欄（content `1;0;10`）＋一個 C-S 欄
  content `N01:80;N02:180;N03:180`，即時顯示「你要付的金額」。
- **markdown 說明區塊（C-M）**：content 放整段 markdown（可含連結
  `[點此進入](https://…)`），用來在題組前插說明文字，欄位名稱就是區塊標題。
- **地址題組**：縣市／鄉鎮市區／路街門牌各一個 `T` 欄（content 用正則把關，如
  `至少要有「號」::號`），最後一個 `P` 欄 content `3;F01,F02,F03` 自動查郵遞區號。
- **認證欄不只主鍵**：type=A 可搭 `format=T`＋正則（如出生年月日
  `格式：「20230101」::^(19|20)\d{2}(0[1-9]|1[0-2])(0[1-9]|[12]\d|3[01])$`），
  登入時逐欄與名冊比對。
- **「無則填無」慣例**：非必填但要格式的欄位，正則寫成 `^(?:09\d{8}|無)$` 一類，
  讓填寫者能明確表達「沒有」。
- **檔案檢視（C-F）發個人化檔案**：content 放資料夾 ID，名冊該欄放每人的檔名
  關鍵字（如身分證號），系統自動找出該人的檔案（如學生照片）。
- **個人化展示文字（C-T）**：名冊值可以是事先算好的整句文字（如
  `你的學年平均：69／年級平均：65.80`），把成績單、興趣量表結果直接餵給每個人看。
  名冊用公式帶入時小心 `#N/A` 這種 Excel 錯誤值會原樣顯示。
- **確認題**：問卷最後放一個必填 `X` 欄（content `100;3;4;4`），要求填寫者
  親手輸入「完全正確」一類字串當送出前的最後確認。
- **欄位 ID 前綴不可信**：實務問卷裡 `A02` 可以是 type C、`S03` 可以是 format X，
  ID 只是識別字串，欄位行為一律以第 3／4 列的 type／format 為準。

## 3. 紀錄表單（record，「新表單ID」指向）

只讀寫第一個分頁。`writeRecord` 送出時 `appendRow`，一列一次送出（同一人重複送出會有多列，
匯出工具取最後一列）。列結構（`pureData`）：

| 欄 | 內容 |
|----|------|
| A | 送出時間（毫秒 timestamp，前端 `writeTick`） |
| B | accept 旗標（`True`） |
| C | 主鍵值（伺服器端 JWT claims 的 pkey，不信前端） |
| D | 簽名圖 Drive fileID，多格簽名以 `;` 分隔（無簽名需求＝空） |
| E | 分組值（type=G format=G 欄位的值，無分組＝空字串） |
| F 起 | 依 headers 順序一欄一值 |

第 1、2 列通常放欄位 ID／名稱當人工對照用標題（F 欄起），程式寫入不依賴它。

### 值的慣例

- `D` 欄位宣告「無資料」時，紀錄該欄的值就是字串「無資料」；`N` 欄位留空落地
  「不提供資料」（2026-07-11 起；更早的紀錄是空字串）——兩個哨兵下游語意不同
  （主動宣告沒有 vs 可空欄位留空），readRecord 回填時「不提供資料」轉回空值、
  「無資料」原樣保留（按鈕狀態靠它）。
- 手機、長數字等欄位值前綴 `📝`（如 `📝0912345678`）：防止 Google Sheets
  自動轉型吃掉前導 0——這是刻意設計，見 `issue.md`，不要「修掉」。
- 檔案上傳欄的值是 Drive fileID；多檔以 `;` 分隔。**這個 fileID 一律由伺服器裁決**
  （Phase 23）：使用者選「沿用上次上傳的檔案」時前端只送哨兵 `__SM_REUSE_LAST_FILE__`，
  writeRecord 從本人紀錄表最後一列查出真 fileID 才落地（**哨兵絕不出現在紀錄表**）；
  新上傳的 fileID 要在 `_file` 登記表對得上（同問卷＋同人＋同欄位）才寫得進來。
- 多選欄的值是 `選項1;選項2` 形式。

### 範例列（虛構）

| A 時間 | B | C 主鍵 | D 簽名fileID | E 分組 | 父親手機 | 購買數量 | 選單 | 多重選單 |
|---|---|---|---|---|---|---|---|---|
| 1686188816972 | True | A123456789 | （空） | 仙人掌班 | 📝0912345678 | 📝6 | 芭樂 | 牛奶;核桃 |
| 1783627864298 | True | A123456789 | 1SigFileIdFakeAAA;1SigFileIdFakeBBB | 仙人掌班 | 📝0912345678 | 6號 | 芭樂 | 核桃;巧克力;杏仁 |

## 4. 暫存試算表（draftSheetID）的分頁一覽

不是「資料」表，是系統自己的內部帳本；**永不對外分享**（`_logins` 存明文帳號、`_invites` 存明文
主鍵值，保護全靠這條紀律）。四張都是純 append、第 1 列凍結表頭、禁 deleteRow：

| 分頁 | 用途 | 欄位 |
|------|------|------|
| `_draft` | 線上暫存草稿（Phase 17/19/20） | A 主鍵**假名**(HMAC id)、B updatedAt(ms)、C referSSID、D 起 payload（`smd1:` 前端加密密文，超長切塊） |
| `_invites` | 遠端簽名邀請快照日誌（Phase 16） | 14 欄，見 struct.md；每格最新列勝出 |
| `_logins` | 登入稽核（Phase 21） | A timestamp(ms)、B referSSID、C **明文真實帳號值**、D 成功/失敗 |
| `_file` | 上傳檔案登記（Phase 23） | A timestamp(ms)、B referSSID、C 上傳者**假名**(HMAC id)、D **欄位 ID**（對照表單第 1 列的 id；不記位置——插欄/搬欄會位移）、E fileID、F mimeType |

`_file` 的用途是讓 writeRecord 能驗證「這個 fileID 真的是這個人在這份問卷的這一欄上傳的」——
沒有它，後端只能原樣相信前端傳來的 fileID（Phase 23 之前的缺口）。只有 metadata，
不含檔案內容；`draftSheetID` 未設時整套暫存/邀請/稽核/登記都靜默停用。
